Architecture

What Powers This System

🍃

Mongoose ODM

Schema-enforced data models with built-in validation, virtuals, and type casting on top of MongoDB Atlas.

🛡️

Data Validation

Every field validated at the schema level. Nigerian-specific enums: JSS1–SS3, blood groups, genotypes, WAEC subjects.

⚡

Error Handling

Try/catch on every route. Meaningful 400/404/500 responses with detailed error messages and field-level feedback.

📊

Dashboard Stats

Live summary endpoint: total students, active enrolments, courses by department, recent registrations.

🔐

Env Secured

MongoDB credentials stored in environment variables. Never committed to GitHub. Render config vars used in production.

📖

Swagger Docs

Full interactive API documentation — test every endpoint directly from the browser without any extra tools.

REST API

All Endpoints

GET /dashboard School overview — totals, breakdowns, recent records
GET /students Retrieve all registered students
GET /students/:id Get a single student by MongoDB ID
GET /students/grade/:grade Filter students by class (e.g. SS2, JSS1)
POST /students Register a new student with full validation
PUT /students/:id Update an existing student record
DELETE /students/:id Remove a student from the system
GET /courses Retrieve all courses/subjects
GET /courses/:id Get a single course by MongoDB ID
POST /courses Create a new course with full validation
PUT /courses/:id Update an existing course
DELETE /courses/:id Remove a course from the system

🎓

This project is part of CSE 341 Web Services at BYU-Idaho, built by GabbyTech. It models a real Nigerian secondary school management system using the JSS1–SS3 / SS1–SS3 grading structure, WAEC subject categories, and Nigerian-specific data fields like genotype and state of origin. The API is designed to eventually connect with the Cinque Stelle CBT platform and serve as a backend component of the broader GabbyTech portfolio of educational tools.
